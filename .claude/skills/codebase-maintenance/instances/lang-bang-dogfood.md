# Checkpoint instance — bang-lang

> Generated by checkpoint/BOOTSTRAP — 2026-06-21; regenerate when gate/docs/conventions move.

A Lean 4 + Mathlib verification kernel for the BANG language: a two-hop verified
compilation (graded-CBPV reference → CalcVM → WasmFX, ADR-0016). The repo's "facts"
are mostly **proofs** (theorems + axiom hygiene) and **docs** (ADRs, roadmap,
volatile context). Phase A is in progress: theorem *statements* frozen, *bodies*
intentionally `sorry` (the burndown is the work-queue, not a failure).

## §1 — The gate

| | command | reads |
|---|---|---|
| **Full gate** (G1+) | `nix develop --command just verify` | real exit code (NOT piped) |
| **Fast per-file** (G0) | `nix develop --command just check Bang/Foo.lean` | single-file elaboration errors |
| **Axiom truth** (G2/G3) | `nix develop --command just axioms` (= `lake env lean Bang/Audit.lean`) | `#print axioms` per headline theorem |
| **Burndown** (status) | `nix develop --command just burndown` | sorry/axiom count per module |

`just verify` = `selfcheck` (zero-dep Node row-unifier check) + `build`
(`lake exe cache get && lake build`) + `audit` (`tools/audit.sh`).

**MUST enter `nix develop` first** — bare `lake`/`just`/`node` are not on PATH.
First cold `build` pulls Mathlib oleans via `lake exe cache get` (network, multi-GB,
minutes). If no network, the gate is "not runnable" — say so, don't fake green.

**Green means:** `lake build` clean + `audit.sh` static guards pass + (for G2/G3)
every headline theorem's axiom set ⊆ `{propext, Classical.choice, Quot.sound}`.
Lint warnings (unused simp args) are present and NON-blocking — not part of green.

### Guards (run around the gate)
- **pre-commit hook** (`tools/git-hooks/pre-commit`, installed via `just install-hooks`):
  1. no `admit` in staged `.lean`;
  2. no hand-rolled `axiom` outside the Spec/Syntax/LR/Operational/Compile/Audit
     family (the Phase-B stub modules);
  3. **statement-change detector** — added/removed `theorem` lines block the commit
     (statements are FROZEN); bypass only with `STATEMENT_CHANGE_OK="why"`;
  4. runs `just verify` IFF `.lean` staged; skip with `BANGLANG_SKIP_VERIFY_REASON="…"`.
- **audit.sh**: lists pending `sorry` (status, non-fail), fails on `opaque` left in
  `Bang/Spec.lean`, runs `lake build`.

### Baseline (known-red the gate tolerates)
- **`sorry` in theorem bodies** — Phase A keeps these on purpose (the Phase B
  burndown queue). Currently ~31 sorry-mentions, ~36 axiom-stubs across `Bang/`.
  Gate on *new* axioms/sorrys beyond this baseline, never on the existing count.
- **Lint warnings** (unused `simp` args, e.g. `CalcReifySim.lean`) — non-blocking.

## §4 — Idiosyncrasies (the project's "what must stay true")

From CLAUDE.md invariants + Do-NOT + pre-commit guards:

1. **Kernel stays at FIVE primitives** (thunk · force · effect rows · handlers · STM).
   A sixth is a spec change requiring an ADR. *Guard: prose only — climb candidate.*
2. **STM is the only privileged kernel primitive.**
3. **Effect rows are sets** — idempotent, union = join; never ordered, never multiset
   (ADR-0001/0018). *Selfcheck exercises the row-unifier; a fitness-function candidate.*
4. **Theorem statements are FROZEN** (the PRD). Filling a body is fine; changing a
   statement is the cardinal sin. *Guard: pre-commit statement-change detector (on test rung).*
5. **Axiom hygiene** — only `{propext, Classical.choice, Quot.sound}` allowed in
   headline-theorem axiom sets; no hand-rolled `axiom` outside Phase-B stub modules.
   *Guard: pre-commit axiom check + `just axioms` (test rung).*
6. **The machine is an OUTPUT of the calculation** — never hand-design the VM then
   justify a compiler against it (ADR-0016).
7. **Proof rides the reference** — anything that runs is `exec` itself or
   differential-tested against it; never an execution path with no oracle.
8. **No implicit lexical capture; no `sig`; reactivity is an operator** (ADR-0005/0006).
9. **Performance is second-class** — don't optimize speculatively.
