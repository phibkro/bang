# Checkpoint instance — occupational-health

> Generated by checkpoint/BOOTSTRAP — 2026-06-21; regenerate when gate/docs/conventions move.

A TypeScript / Effect-4 / Cloudflare monorepo (pnpm workspace, Vite+ `vp` toolchain):
sensor-agnostic occupational-health platform for Norwegian workplaces. RPC-first API,
hexagonal DDD bounded contexts, a content-addressed event-ledger spine on Cloudflare
Workers + Durable Objects. The repo already runs an unusually strong maintenance stack
(doc-path link-check, import-boundary fitness function, pattern lint) — most of the
nine surfaces are already on the *test* rung, not survey.

SoT for everything below: `CLAUDE.md` (its "Verification Gates", "Definition of Done",
"Architecture", "API + Auth" sections). The instance points at it; it does not copy it.

## 1. The gate

**Canonical full gate:** `scripts/verify.ts` — the command wired into the Claude
`Stop` / `SubagentStop` hooks and named as the gate in CLAUDE.md "Definition of Done".
It runs, in fail-fast order:

```
0. node scripts/check-docs.mjs        # doc-drift: backtick repo-paths in CLAUDE.md/TODO.md must exist
1. vp check  (per dependency tier)    # tsc + eslint(functional/effect) per package, core→domain→infra→api
2. depcruise --config .dependency-cruiser.cjs packages/ apps/   # import-boundary fitness function
3. scripts/check-patterns.ts          # declarative regex lint (no-raw-typed-sql, no-as-assertion)
4. vp test --run --passWithNoTests    # domain + infra + api + worker tests, per package
```

Real-exit-code form (the gate emits JSON `{continue:true|false}` and `process.exit(0)`
always — so DO NOT trust its process exit; read the `continue` field, or invoke the
underlying tools directly for a real exit code):

```bash
nix develop                                    # ENTER FIRST — vp/node live here
# but note: `bun` is NOT in the dev shell or node_modules (see GOTCHA below)
node scripts/check-docs.mjs ; echo $?          # SAFE, offline, real exit code — runs anywhere
PATH="$PWD/node_modules/.bin:$PATH" vp check    # full per-tier tsc+lint, real exit code
```

> **GOTCHA — the gate's shebang is `#!/usr/bin/env bun`, but `bun` is not installed**
> (not in the flake dev shell, not in `node_modules/.bin`). `vp` lives only in
> `node_modules/.bin/vp` (the `vite-plus` devDep), NOT on the dev-shell PATH. So
> `scripts/verify.ts` as written needs bun + a populated `node_modules`. To run the
> gate you either `bun scripts/verify.ts` (needs bun) or run the four tiers by hand
> with `node_modules/.bin` on PATH. CLAUDE.md says "bun as CLI runtime" — confirm bun
> is provisioned before trusting the one-shot gate. This is a known field gotcha for
> bang-lang too (`nix develop` first; bare tools not on PATH).

**Fast per-file loop (G0/G1):**
- `vp check --fix <file>` — the PostToolUse hook (`.claude/settings.json`) already runs
  this on every Write/Edit of a `.ts/.tsx/.md/.json/...` file. Format + lint + typecheck.
- `vp check --quick` analogue: `bun scripts/verify.ts --quick` skips tests (tiers 0-3 only).
- `node scripts/check-docs.mjs` — zero-dep, offline, instant; the cheapest single check.

**Pre-commit hook** (`.githooks/pre-commit`, a generic havn auto-detect hook): format
(always) + lint+build (always) + tests + changeset (main only). NOTE: it auto-detects
turbo/bun/npm and `bunx changeset` — but this repo has **no `.changeset/` dir, no
turbo.json, no `bun`**, so several branches no-op. The *real* enforced gate is the
Claude-hook `scripts/verify.ts`, not this pre-commit hook.

**Baseline (gate on NEW violations only):** 3 `@occupational-health/core` test files
fail to LOAD on the `registry ↔ core` workspace cycle ("Cycle dependency detected" under
`vp run test -r`). Pre-existing — run tests **per-package** (`cd packages/<pkg> && vp test
--run`) to avoid it. Only *new* failures count. (CLAUDE.md "Test Baseline" + "Definition
of Done".)

**Guards bundled in the gate:**
| guard | rung | what it catches |
|---|---|---|
| `check-docs.mjs` | test | dead backtick repo-paths in CLAUDE.md / TODO.md |
| `dependency-cruiser` | test | cross-package import-boundary violations (8 forbidden rules) |
| `check-patterns.ts` | test | `no-raw-typed-sql`, `no-as-assertion` regex rules |
| `vp check` eslint | test | `no-let`/`no-throw`/`no-loop`, `consistent-type-assertions:never` (no `as T`) |
| Effect LSP | test | `schemaSyncInEffect` (no `decodeSync` in `Effect.gen`), `globalErrorInEffectFailure` |

## 3. Work-type → survey table

Conventional-commit prefixes with package scopes (`feat(api)`, `fix(api)`, `docs(adr)`,
`refactor(core)`, `test(api)`, `chore`, `docs(todo)`, `docs(claude)`). Scope = which
package(s) the change touched → which import-boundary + which doc.

| work-type | tends to invalidate → confirm/update |
|---|---|
| `feat(api)` / RPC command added/changed | CLAUDE.md "API + Auth" (the 55-command count, the `commandPolicy` completeness), the authz matrix, `docs/prd/*` if domain behavior, the **Test Baseline** count, possibly a new ADR |
| `feat(worker)` / ledger / CF resource | `docs/ledger-spine-overview.md`, CLAUDE.md "Stack" CF-wiring recipe, `TODO.md` Shipped, `oh-ledger-spine` memory, D1 read-model version (`exposure_projection_vN`) |
| `feat(web)` | CLAUDE.md `apps/web/` monorepo note (which pages read live worker vs in-memory), E2E spec list, Playwright count in "Test Baseline" |
| `fix(api)` | usually no doc; a Test Baseline bump if a new test; an ADR only if it reveals a settled structural choice (e.g. the `firewall` fix referenced ADR-0012) |
| `refactor(<pkg>)` | import-boundary (depcruise) must stay green; "test the contract not the implementation" — tests should NOT break; no doc unless a public RPC/schema shape moved |
| `docs(adr)` | a NEW `docs/decisions/00NN-*.md` (next number); cross-link from CLAUDE.md; supersede-link integrity if it replaces one |
| `feat` touching a hard-to-reverse choice | **write an ADR** (Definition of Done makes this mandatory), update CLAUDE.md rule line |
| `chore` / rename | G0/G1 only — skip survey unless it moves a backtick path in CLAUDE.md/TODO.md (then check-docs catches it) |
| any deferral | **must be written to `TODO.md`** (Definition of Done: "Deferred items written down, not dropped silently") |

## 4. Idiosyncrasies — the non-obvious "what must stay true"

From CLAUDE.md "Rules" / "Architecture" / "API + Auth" / "Migrations":

1. **Norwegian regulatory limits, NOT EU.** Never use EU directive values; verify against
   Lovdata. Stricter is correct. (Domain-correctness invariant — a survey can't autocheck
   this; it's the highest-stakes semantic fact in the repo.)
2. **`Schema.DateTimeUtcFromString`, never bare `Schema.DateTimeUtc`.** In Effect 4 the
   names INVERTED vs v3 — bare is now the FromSelf variant and breaks the JSON-wire
   round-trip. **`tsc` does NOT catch it** — only a decode/round-trip test does. (So:
   every Schema needs a round-trip test; this is in "Definition of Done" implicitly via
   "Every Schema gets a round-trip test".)
3. **Never `decodeSync` inside `Effect.gen`** — use `Schema.decodeEffect` (typed error
   channel) or `.pipe(Effect.orDie)`. Enforced by Effect LSP, not `vp check` alone.
4. **No `as T` assertions** in domain code (`consistent-type-assertions:never` + the
   `no-as-assertion` pattern rule); `as const` allowed.
5. **Import boundaries are structural** (depcruise): `core` is a leaf; `core`/`infrastructure`
   never import from `apps/`; each output-boundary context imports only `core` (+`ingestion`
   for events). 8 forbidden rules in `.dependency-cruiser.cjs`.
6. **RPC command count + authz completeness.** 55 RPC commands across 7 contexts; the
   `commandPolicy` map must classify ALL of them — a completeness test keeps it in lockstep
   with the live group. Adding an RPC command without classifying it breaks the authz test.
7. **Role comes from the registry, NOT the token.** (ADR-0012 invariant.) Auth fails closed,
   leaks no reason. `AUTH_MODE=demo` permissive (X-Demo-Role header) vs `enforced` (real crypto).
8. **HazardDomain values ALWAYS uppercase** (`"NOISE"`,`"DUST"`,`"HAV"`,`"WBV"`,`"GAS"`);
   strict equality in all filters. `alpha_quartz` in code, "α-kvarts" in prose.
9. **ASCII-only code identifiers.** Norwegian terms in prose only.
10. **Single consolidated migration** (`0001_initial.sql`) — NO incremental migrations
    pre-production (PGlite can't do ALTER/RENAME across files). ADR-0006. A "feature" that
    adds a migration file violates this until production data exists.
11. **Events are facts** — immutable, past-tense, published after the fact, never for control
    flow between contexts. Contexts communicate via events (outbox) + queries, never shared state.
12. **ADR discipline:** settled hard-to-reverse decision → a new dated ADR in `docs/decisions/`
    (format: `# ADR NNNN: title` / `**Date**` / `**Status**` / Context / Decision / Consequence).
    Status `Accepted`. (No README index in the dir — see climb.)

| 8 | **architecture** | yes | **test** | `dependency-cruiser` (8 forbidden boundary rules) + `check-patterns.ts` are exactly fitness functions. Strong. The "core is privileged leaf" invariant is enforced, not prose. Best-in-class surface for this repo. |
| 9 | **decisions** | yes (12 ADRs) | **survey** | Dated, with Context/Decision/Consequence + rejected-alternatives. **Top 3 climbs (ranked):**
1. **`gitleaks` secret-scan** (surface 7) — cheapest, highest-consequence missing gate; the
   repo is auth-heavy and a leaked secret is unrecoverable.
2. **`--frozen-lockfile` in CI** (surface 4) — Effect-4-beta caret-breaks make lockfile↔manifest
   drift uniquely dangerous here; a cached green build could hide it.
3. **Generate / AST-anchor the count-prose in CLAUDE.md** (surface 3) — "55 commands", "~380
   tests", the authz matrix are hand-maintained and *will* drift; the `commandPolicy` completeness
   test already proves the count is derivable — surface it as a generated number.
