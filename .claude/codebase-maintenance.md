This repo sits high on the derivation ladder (`generate > test > survey`, defined in the global
skill) — most derived facts are *generated* or *tested*, so the hand-survey is small. The
maintenance instinct: when a NEW derived fact appears, climb it; don't let it land on survey.

### Read the gate's real result via a format-independent signal, never a proxy

The rule is generic (gate the build artifact via **exit-code / forced rebuild**, never a grep,
cached, or piped proxy) — homed in the **`/gate`** skill + memories `gate-sorries-via-axioms-not-grep`
and `nix-build-verify-exit-codes`. Every proxy below has produced a false green here; the
repo-specific instantiations:

- **`grep "error:"` misses `error(lean.unknownIdentifier):`** — this Lean prints name-resolution
  errors (forward-ref / unknown identifier) as `error(category):` in some environments, `error: …`
  in others (a diagnostic-format split across machines/versions). Gate with `grep -E "error"` (no
  colon — catches both) or `lake build`'s exit code (format-independent). (Same rule in CLAUDE.md
  §How-to-verify.)
- **Piped exit** — `lake build … | head` returns head's `0`. Use the unpiped exit / `$PIPESTATUS[0]`
  / a store-path check. (memory `nix-build-verify-exit-codes`.)
- **Stale `.olean`** — `#print axioms` reads the *cached* olean, which can predate a worktree's
  checked-out source (each worktree has its own `.lake`). The **tell**: an inconsistency impossible
  under clean elaboration — a caller showing a *smaller* axiom set than a callee it invokes. Fix:
  `touch <file> && lake build <Module>` (force a rebuild) BEFORE the `#print axioms`.

**Gate the committed content on a clean tree**, never a dirty working tree or an agent's summary —
a green build with a hidden `sorryAx` is a false done. (CLAUDE.md §How-to-verify + memory
`gate-committed-content-not-worktree`.)

**Hook escapes.** The pre-commit hook runs `just fitness` (fast, no-build) on EVERY commit, then
`just verify` (the build) for `.lean` commits. When the loogle dep's periodic re-clone breaks the
build leg on an unrelated network/tree issue, escape it with `BANGLANG_SKIP_VERIFY_REASON="…" git
commit` — **fitness still gates** — then run the full `nix develop -c lake build` yourself before
trusting green. `git commit --no-verify` is the HARD escape that skips fitness too — emergencies
only. The whole gate is read-only (no mutating apply/deploy step), so it's safe at every checkpoint.

**In-place enforcement.** The #40 cache-get-in-worktree footgun is a STRUCTURAL guard —
`tools/hooks/pretool-gate-guard.sh` (PreToolUse, in `.claude/settings.json`) denies `lake exe cache
get` from a linked worktree (detected by `git-dir ≠ git-common-dir`, not a path string). The
axiom-gate is the **`/gate`** skill; read-only refute-first verification is the **`lean-proof-auditor`**
agent. The grep-traps above stay GUIDANCE, not a hook block — a hook can only string-match the
command, which false-positives on the pattern quoted inside a message: enforce the unambiguous,
guide the heuristic.

## Debt prevention by construction

Debt is prevented *before creation* by refusing to let a derived fact land on the survey rung when
it could be generated or tested. When you add X, ask: **"what fact does X derive, and which rung
enforces it?"** — then climb:

- a new invariant in prose → a fitness function (like `check-primitives.sh`), not a comment.
- a new derived doc/index → generate it + a `--check` in the gate (the ADR ledger is the worked
  example: index + Q-status + Status are all generated/tested from ADR frontmatter, so they CANNOT
  disagree — the SSoT debt is made unrepresentable, not detected).
- a new theorem → its axiom set is auto-audited; a new sorry → document + track it (Spec.lean
  header + burndown) or it fails the audit.
- a new cross-reference between two docs → a cross-ref test (like Q⟺ADR), not "remember to sync".

Each debt category in this repo (proof · trust · oracle · SSoT · frozen-statement · kernel-creep ·
doc-drift · worktree-sprawl) maps to a maintenance-object row above and the guard that makes it
unrepresentable; the instinct is the same across all of them.

## Worktree teardown safety (the designated SoT — memory `parallel-agent-writes-need-worktrees` homes this rule here)

Worktree cleanup is a **destructive op**, and the multi-agent setup makes "is this worktree done?"
genuinely hard (agents can't be reliably stopped; idle ≠ stopped; stand-down messages are async). A
force-remove once discarded ~1hr of a live writer's uncommitted WIP. The rule, structural not
remembered:

```
1. NEVER blanket `--force` a destructive git op (worktree remove, branch -D, reset --hard, prune,
   gc --prune=now). Default to the NON-forcing form — `git worktree remove` REFUSES on a dirty tree,
   which is the fail-loud guardrail. Force ONE target only after `git status` on THAT worktree confirms
   its dirtiness is expendable.
2. Teardown-safety = working-tree CLEAN (`git status`) + writer CONFIRMED-STOPPED. NOT committed-
   subsumption (`git rev-list ^trunk = 0`) — that answers "are the commits redundant", and says NOTHING
   about uncommitted work or whether a writer is live. Uncommitted WIP is invisible to rev-list.
3. If a worktree has uncommitted changes, STOP — assume a live writer. Removing it silently discards
   their work.
```

The friction of a command that *refuses* is the safety feature — don't suppress it. (Cross-project
memory: `parallel-agent-writes-need-worktrees`.)

## Grades

(Generic G0–G3 definitions: global skill §"Run — grade by seam". The per-repo bindings:)

- **G0** (per-edit): `just check FILE` — fast single-file Lean error check.
- **G1** (per-commit, hook): `STATEMENT_CHANGE_OK` guard, then `just fitness` (no-build) on EVERY
  commit — so docs-only and build-skipped commits are gated too; `just verify` (the full build) runs
  for `.lean` commits unless `BANGLANG_SKIP_VERIFY_REASON`. Tree committed.
- **G2** (feature / checkpoint / wrap): G1 + hand-survey the **prose status** of CONTEXT/ROADMAP —
  the only genuine survey residue (PATHs reachability, refs, SHA waypoints, and proof-state are
  already in `just fitness`).
- **G3** (release / ◊ close): full sweep + `just axioms` per headline + cold-agent dogfood (a fresh
  agent reading CLAUDE.md + CONTEXT.md + `git log` orients correctly).

## Anti-triggers

A pure rename, a comment typo, a `docs:` commit with no status/ADR/frozen-statement surface →
G0/G1, skip the survey. A proof-only commit that doesn't touch a frozen statement or add a sorry →
the axiom gate is the whole survey. (Generic: global skill §Anti-triggers — scale the pass to the
change; surveying nothing is correct when the blast radius is empty.)
